## 2. 识别出的缺失与优化点 (Identified Issues)

1.  **[UX] 操作反馈缺失**: 心情签到点击“完成”后直接返回，缺乏成功的视觉或震动反馈。
2.  **[UX] 导出进度反馈**: 导出大数据量时（虽然目前限制30天），界面无 Loading 状态。
3.  **[健壮性] SwiftData 异常处理**: 虽然模型简单，但未显式处理磁盘空间不足等导致的保存失败。
4.  **[无障碍] VoiceOver 支持**: 图表组件和表情按钮对视障用户的描述信息不完善。
5.  **[国际化] 硬编码字符串**: 部分 UI 字符串仍为硬编码，未完全使用 `LocalizedStringKey`。

---

## 3. 优先级排序的完善计划 (Improvement Plan)

### P0: 健壮性与无障碍 (安全性/核心体验)
- **任务 1**: 为所有 SwiftData 存储操作添加错误捕获与用户提示。
- **任务 2**: 为表情按钮和图表添加 `accessibilityLabel`。

### P1: 用户体验增强
- **任务 3**: 引入 `Haptics` 触感反馈（签到成功、导出完成）。
- **任务 4**: 优化导出流程，添加交互式 Loading 状态。

### P2: 扩展性与规范
- **任务 5**: 全面清理硬编码字符串，准备支持多语言切换。

## 4. 技术方案与测试用例

### 技术方案：触感反馈 (Haptics)
- **实现**: 使用 `UIImpactFeedbackGenerator`。
- **场景**: 心情签到成功 (Medium)、安全签到成功 (Success)、数据清除 (Warning)。

### 测试用例：无障碍验证
- **用例 ID**: TC-ACC-01
- **步骤**: 开启系统 VoiceOver -> 导航至心情签到 -> 选中表情按钮。
- **预期**: 语音播报“悲伤，表情按钮”或“非常开心，表情按钮”，而非仅播报“表情”。

### 性能基准
- **启动时间**: < 1.1s (冷启动)
- **内存占用**: 稳态 < 60MB, 峰值 < 90MB
- **渲染**: 图表滑动与切换 60fps 无掉帧。
